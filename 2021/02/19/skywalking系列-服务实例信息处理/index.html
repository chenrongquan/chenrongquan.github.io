skywalking系列 | 服务实例信息处理

2021-02-19

Agent采集端 ServiceManagementClient

服务及实例信息上送

  • prepare()方法

    • 将自身注册到GRPCChannelManager来监听网络连接
    • 生成当前InstanceName=UUID@ip 或者 Config.Agent.INSTANCE_NAME指定

  • boot()方法

    • 启动一个定时任务线程，实现定时服务注册与心跳检测；默认每隔30秒进行一次检测，也可由Config.Collector.HEARTBEAT_PERIOD指定

  • statusChanged()方法

    • 网络连接创建成功时，会依赖该连接创建stub客户端

===30s后===

  • 定时任务run()方法
    • 判断Channel连接状态
    • 每第10次调用(即30*10s)进行一次服务及实例的注册及更新
    • 每次调用进行心跳检测(不包含每第10次)

OAP接收端 ManagementServiceHandler

实例注册 reportInstanceProperties()方法

  • 上送数据: InstanceProperties

    心跳检测 keepAlive()方法

  • 首次注册30秒后 上送数据

注：

  1. 落 instance_traffic 时机：首次注册
  2. 落 service_traffic 时机: 首次心跳检测

流计算处理

以实例信息为例

数据源处理

  • 数据源Source：ServiceInstanceUpdate
  • 类型转换Dispatcher：InstanceTrafficDispatcher
  • 指标数据Metrics：InstanceTraffic 流计算可接受的原始数据类型

流计算模型

  • 指标模型：MetricsStreamProcessor 针对Metrics数据
  • 处理逻辑
  1. 阶段L1: 针对相同的ID的实体，在当前OAP节点进行汇集计算
  2. 阶段L2: 通过ID进行hash路由，在分布式环境中，对离散数据进行二次汇集；在通过与存储中的数据的汇集计算，完成入库。

L1数据接收和解析

  • 职责：数据接收和解析，并进行当前OAP节点内的数据聚合，使用OAL或者其他聚合模式；对应阶段L1
  • MetricsAggregateWorker 采用内存队列dataCarrier, MetricsAggregateWorker.instanceName
    • 数据接收 in()方法 对应 入队
    • 数据解析 onWorker()方法 accept()数据处理，规则 InstanceTraffic#combine()方法
    • 数据发送L2 nextWorker.in()

      规则

      1. 更新 lastPingTimestamp
      2. 若 properties 不为空 更新 properties
      3. 保持 timeBucket 为注册时间

L2分布式聚合

  • 职责：分布式聚合，根据一定的路由规则，将经过Receiver聚合后的数据路由到指定节点后，进行二次汇集，并入库。 对应阶段L2
  • 使用 hash select 路由策略，根据计算的实体的hash值来选择对应的OAP Server
  • MetricsPersistentWorker 采用内存队列dataCarrier, MetricsPersistentWorker.instanceName
    • 数据接收 in()方法
    • 数据解析 onWorker()方法 数据写入缓存，二次汇集 MergableBufferedData#accept() 规则同L1

存储模型

存储定时器 PersistenceTimer

  • 启动定时任务 每隔5秒进行批量存储

  • 数据组装buildBatchRequests()方法，读取缓存，批量保存

  • MetricsPersistentWorker#prepareBatch 批量落库

    • 批量大小上线 2000 硬编码

  • 落库flushDataToStorage()

    • 查库，MetricsEs7DAO#multiGet()
    • 比较 缓存Metrics 和 库中 noInCacheMetrics 比较规则 InstanceTraffic @EqualsAndHashCode serviceId+name
    • 新增 prepareBatchInsert
    • 更新 combine处理后更新 prepareBatchUpdate
